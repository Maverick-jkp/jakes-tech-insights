# Keyword Strategy Guide

## 핵심 원칙

> **트렌드는 자동화의 입력값이 아니라, 사람이 해석해야 하는 원재료다.**
>
> 자동화는 '글을 쓰는 데' 쓰고, 키워드 선택엔 반드시 인간의 마지막 필터가 필요하다.

## 왜 "핫한 트렌드 키워드 자동 수집"은 실패하는가

### ❌ 위험한 접근
- Google Trends → 바로 글 생성
- Reddit Hot → 그대로 글화
- "오늘 뜨는 키워드 Top 10" → 자동 포스팅

### 문제점
1. **경쟁 과다**: 대형 미디어, SEO 전문 블로그, 애드센스 팜, AI 자동 생성 사이트가 이미 점령
2. **자동화 냄새**: 제목 패턴, 구조, 문장 톤이 비슷해져서 Google과 사람 모두 예민하게 반응
3. **정보 소비로 끝남**: 트렌드 키워드는 한 번 읽고 끝 → 체류 시간 짧음 + 재방문 없음 + 브랜드 기억 안 남음

## ✅ 우리가 취할 전략

### 핵심 공식
```
[트렌드] × [시간 지연] × [해석/의사결정 관점]
```

**지금 뜨는 것이 아니라, 조금 지나서 사람들이 "결정하려고 검색하는 것"**

### 노리는 검색 의도
- "이거 써야 하나?"
- "계속 가도 되나?"
- "대안은 뭐지?"
- "실제론 어떤가?"

👉 **이 검색은 경쟁이 적고 체류 시간이 길고 재방문 확률이 높음**

## 4가지 키워드 타입

### 1. 트렌드 + 선택 단계 키워드 (최우선)

❌ **흔한 키워드**
- AI coding assistant 2025
- 리모트워크 트렌드
- 스타트업 투자 동향

✅ **한끗 다른 키워드**
- "AI coding assistant 실제로 써보니, 언제 비효율적인가"
- "리모트워크 툴, 6개월 써보고 결국 남는 것"
- "2025년 투자 트렌드 중 개인이 조심해야 할 착시"

### 2. 트렌드의 부작용 / 한계 / 실패 케이스

자동화 블로그가 거의 안 건드리는 영역:
- "Cursor AI 쓰다 팀에서 다시 GitHub Copilot로 돌아간 이유"
- "요즘 스타트업들이 투자 받기 더 어려워진 진짜 이유"
- "리모트워크 툴이 많아질수록 생산성이 떨어지는 순간"

### 3. 언어/지역 관점 차이 (우리 강점)

멀티언어 능력 활용:
- "미국에서 말하는 AI productivity와 일본에서의 의미 차이"
- "한국 스타트업 투자 트렌드를 영어로 보면 오해하는 포인트"
- "일본 리모트워크 문화에서 해외 툴이 실패하는 이유"

### 4. 후행 키워드 (1-3개월 뒤)

Google Trends에서 **상승 초반 말고 피크 지나고 하락 시작할 때**를 노림:
- "이거 계속 써도 되나?"
- "대안 뭐 있지?"
- "정리된 글 없나?"

## Claude 키워드 큐레이션 프롬프트

```
역할:
너는 SEO 자동화 블로그를 위한 키워드 전략가다.
단, 일반적인 트렌드 키워드는 제외한다.

목표:
한국어 / 영어 / 일본어 각각에서
"결정 단계(Decision stage)"에 있는 사용자가
검색할 가능성이 높은 키워드만 제안하라.

금지:
- 단순 트렌드 요약 키워드
- 뉴스성 키워드
- 이미 대형 미디어가 점령한 키워드
- "2025 트렌드", "최신 동향" 같은 표현

우선순위 키워드 유형:
1. 트렌드 + 실제 사용 후 판단 (pros/cons, 언제 비효율적인가)
2. 트렌드의 한계, 실패 사례, 오해
3. 비교 키워드 (A vs B) 중 "결정 포인트" 중심
4. 지역/언어/문화 차이 관점
5. 1–3개월 후 검색될 가능성이 높은 후행 키워드

출력 형식:
언어별로 5개씩 제안하라.

각 키워드마다 반드시 포함:
- keyword
- language (ko / en / ja)
- search_intent (왜 검색하는지 한 문장)
- angle (이 키워드를 다룰 때의 관점)
- competition_level (low / medium / high 중 추정)
- why_it_works (이 키워드가 자동화 블로그에 적합한 이유)

예시는 절대 사용하지 말고,
지금 시점에서 현실적인 키워드만 제안하라.
```

## Topic Queue 설계

### 구조: Trend vs Evergreen 분리

```json
{
  "trend": {
    "pending": [],
    "completed": []
  },
  "evergreen": {
    "pending": [],
    "completed": []
  }
}
```

### Trend 큐 스키마 (유통기한 있음)

```json
{
  "id": "trend-2026-01-ai-coding-inefficiency",
  "keyword": "AI coding assistant 언제 비효율적인가",
  "language": "ko",
  "category": "tech",
  "priority": 5,
  "trend_source": "HN + Twitter",
  "angle": "과도한 자동화의 한계",
  "expiry_days": 21,
  "created_at": "2026-01-17"
}
```

**특징**:
- `expiry_days` 있음
- 타이밍 지나면 자동 폐기 가능
- 짧고 날카로운 글에 적합

### Evergreen 큐 스키마 (자산형)

```json
{
  "id": "evergreen-remote-work-tool-selection",
  "keyword": "리모트워크 툴 선택 시 가장 많이 실패하는 이유",
  "language": "ko",
  "category": "business",
  "priority": 3,
  "angle": "의사결정 실수 방지",
  "updated_at": "2026-01-17"
}
```

**특징**:
- 유통기한 없음
- 업데이트 가능
- SEO 누적형

## 운영 가이드

### 주간 비율 추천
```
Trend : Evergreen = 3 : 7
```

**중요**: 자동화 블로그라도
- "전부 트렌드" → 금방 죽음
- "전부 에버그린" → 성장 느림

### Week 단위 루틴 (5분 투자)

1. **Claude로 "한끗 다른 키워드" 15개 생성**
2. **너가 5분 동안 필터링**:
   - 자동화 냄새 나는 것 제거
   - "내가 읽을 만한가?" 기준으로 선별
3. **Trend 3개 / Evergreen 2개 큐에 추가**
4. **파이프라인은 그대로 자동 생성**

### 워크플로우: 수동 실행 (추천)

```bash
# 주 1회 (예: 일요일 저녁) 실행
python scripts/keyword_curator.py

# 출력:
"""
=== 이번 주 키워드 후보 15개 ===

[Tech - English]

2. "AI coding tools이 팀 생산성을 오히려 떨어뜨린 순간"

[Business - Korean]
3. "2025년 스타트업 투자, 미국 vs 한국 착시 포인트"
4. "리모트워크 툴 5개 써보고 남은 건 2개"

...

=== 어떤 키워드를 큐에 추가할까요? ===
선택 (쉼표로 구분, 예: 1,3,5): _
"""
```

**장점**:
- 주 1회 5분 투자
- 품질 관리 가능
- 알람 피로 없음

## 금지 사항

### ❌ 절대 하지 말 것
- Google Trends 상위 키워드 그대로 사용
- Reddit/HackerNews Hot 키워드 자동 수집 → 바로 생성
- "핫한 키워드" 리스트 자동화
- 단순 트렌드 요약 글
- 뉴스성 콘텐츠
- 시간 민감한 속보성 키워드

### ✅ 대신 할 것
- 트렌드를 **소재**로만 사용
- **결정 단계** 검색 의도에 집중
- **해석/관점** 추가
- **인간 필터링** 5분 투자

## 참고 자료

- 이 전략은 **ChatGPT 분석 + 실전 SEO 경험**을 바탕으로 작성됨
- 자동화 블로그가 실패하는 이유: 자동화 자체가 아니라 **사람의 마지막 필터 부재**
- 성공하는 자동화 블로그: **무엇을 쓰느냐**를 잘 고름
